# 네이버 블로그 자동 포스터

## 목적

네이버 블로그에 글을 **자동으로 발행**하는 도구.
- 아이폰 Safari에서 웹 UI로 즉시 포스팅
- 매일 오전 11시 예약 글 자동 발행
- 제목, 본문, 카테고리, 태그, 사진 모두 지원

## 왜 만들었나

네이버는 블로그 글쓰기 API를 **2024년에 폐기**했다.
개발자센터에 "블로그" 항목 자체가 없고, 블로그 관리에도 API 설정 메뉴가 없다.
유일한 방법은 **Selenium 브라우저 자동화**.

## 시스템 구조

```
┌─────────────────────────┐
│   iPhone Safari          │
│   (반응형 웹 UI)          │
│                          │
│   제목 / 본문 / 사진 입력  │
│   [즉시 발행] [예약 추가]  │
└──────────┬───────────────┘
           │ HTTP
           ▼
┌─────────────────────────────────┐
│   FastAPI 서버 (localhost:8000)  │
│                                 │
│   POST /api/post    즉시 발행    │
│   POST /api/post    예약 큐 추가 │
│   DELETE /api/queue/{id}  삭제   │
│   POST /api/login-test  테스트   │
│                                 │
│   ┌───────────┐ ┌─────────────┐ │
│   │ Scheduler │ │  SQLite DB  │ │
│   │ 매일 11시  │ │  글 큐 저장  │ │
│   └─────┬─────┘ └─────────────┘ │
│         │                       │
│         ▼                       │
│   Selenium + Chrome             │
│   → 네이버 로그인 (쿠키 유지)     │
│   → 블로그 글쓰기 페이지 조작     │
│   → 발행                        │
└─────────────────────────────────┘
```

## 사용 시나리오

### 시나리오 1: 아이폰에서 즉시 포스팅

```
1. Safari에서 http://서버주소:8000 접속
2. 제목 입력: "강남 맛집 추천 베스트 5"
3. 카테고리: "맛집 리뷰"
4. 태그: "강남맛집, 맛집추천, 강남역"
5. 본문 작성
6. 📷 사진 3장 추가 (카메라롤에서 선택)
7. [즉시 발행] 버튼 클릭
8. → 서버가 Selenium으로 네이버 블로그에 발행
9. "발행 완료!" 토스트 메시지 확인
```

### 시나리오 2: 글 미리 써두고 매일 자동 발행

```
1. 주말에 글 5개를 미리 작성
2. 각 글마다 [예약 큐 추가] 버튼 클릭
3. → SQLite에 5개 글 저장

4. 월~금 매일 오전 11:00
5. → 스케줄러가 큐에서 가장 오래된 글 1개를 꺼냄
6. → Selenium으로 네이버 블로그에 자동 발행
7. → 발행 완료 표시
```

### 시나리오 3: 최초 설정

```
1. .env 파일에 네이버 ID/PW 설정
2. 서버 실행: python main.py
3. 웹 UI 접속 → [로그인 테스트] 클릭
4. Chrome 창이 뜨면서 네이버 자동 로그인
5. 캡챠 나오면 → 수동으로 1회 풀어주기
6. → 이후 Chrome 프로필에 쿠키 저장되어 자동 로그인 유지
```

## 파일 구조

```
naver-blog-poster/
├── main.py              # FastAPI 서버 + 스케줄러 + API
├── naver_blog.py        # Selenium으로 네이버 블로그 조작
├── models.py            # SQLite DB 모델 (예약 글 큐)
├── config.py            # 환경변수 설정
├── templates/
│   └── index.html       # 웹 UI (아이폰 반응형)
├── static/
│   └── style.css        # 스타일
├── uploads/             # 업로드된 이미지 저장
├── chrome_profile/      # Chrome 로그인 세션 유지
├── requirements.txt     # Python 패키지
├── .env                 # 네이버 계정 정보 (git 제외)
└── README.md            # 이 파일
```

## 기술 스택

| 기술 | 용도 |
|------|------|
| **FastAPI** | 웹 서버 + REST API |
| **Selenium** | 네이버 브라우저 자동화 |
| **APScheduler** | 매일 11시 크론 스케줄링 |
| **SQLite + SQLAlchemy** | 예약 글 큐 저장 |
| **Jinja2** | 웹 UI 템플릿 |
| **pyperclip** | 로그인 시 복사-붙여넣기 (캡챠 우회) |

## 실행 방법

```bash
# 1. 패키지 설치
pip install -r requirements.txt

# 2. 환경변수 설정
cp .env.example .env
# .env 파일에 네이버 ID/PW 입력

# 3. 서버 실행
python main.py

# 4. 브라우저 접속
# http://localhost:8000
```

## 주의사항

- **최초 1회 수동 로그인 필요**: 캡챠가 나오면 직접 풀어야 함. 이후 쿠키로 자동 로그인.
- **하루 1~2회 적정**: 대량 포스팅은 블로그 저품질 판정 위험.
- **네이버 UI 변경 시**: CSS 셀렉터 수정 필요 (naver_blog.py).
- **서버가 꺼지면 스케줄러 중단**: 항상 켜둬야 함 (클라우드 배포 권장).
